## 1. Lưu Ý Kỹ Thuật Quan Trọng

### 1.1 Overlap Detection — Data-Driven

**Đề xuất:** Chuyển sang **data-driven overlap** trong `BoardModel` (pure C#):

```csharp
public class BoardModel {
    public void UpdateSelectableStatus() {
        foreach (var tile in _tiles.Where(t => t.State == TileState.InBoard)) {
            bool isCovered = _tiles.Any(other =>
                other.State == TileState.InBoard &&
                other.LayerIndex < tile.LayerIndex &&
                RectsOverlap(tile.GridPosition, other.GridPosition)
            );
            tile.IsSelectable = !isCovered;
        }
    }

    private bool RectsOverlap(Vector2Int posA, Vector2Int posB) {
        return Mathf.Abs(posA.x - posB.x) < tileWidth &&
               Mathf.Abs(posA.y - posB.y) < tileHeight;
    }
}
```

**Ưu điểm:**
- ✅ Không cần Physics system
- ✅ Unit test không cần Unity
- ✅ Deterministic

### 1.2 DOTween Best Practices

```csharp
// ✅ ĐÚNG: Track tween + kill trong OnDestroy
public class TileView : MonoBehaviour {
    private Tween _currentTween;

    public Tween AnimateMove(Vector3 target, float dur) {
        _currentTween?.Kill();
        _currentTween = transform.DOMove(target, dur)
            .SetAutoKill(true)
            .OnComplete(() => _currentTween = null);
        return _currentTween;
    }

    private void OnDestroy() {
        _currentTween?.Kill();
    }
}

// ❌ SAI: Orphaned tween
transform.DOMove(...);
```

### 1.3 Event Channel — Tránh Memory Leak

```csharp
// ✅ ĐÚNG: Subscribe trong OnEnable, Unsubscribe trong OnDisable
[SerializeField] private TileSelectedChannelSO _tileSelectedChannel;

private void OnEnable() {
    _tileSelectedChannel.Subscribe(OnTileSelected);
}

private void OnDisable() {
    _tileSelectedChannel.Unsubscribe(OnTileSelected);
}

// ❌ SAI: Subscribe anonymous lambda
_tileSelectedChannel.Subscribe(data => DoSomething(data));
// → Không thể Unsubscribe!
```

### 1.5 Async/Await với DOTween

```csharp
private async void OnTileSelected(TileSelectedEventData data) {
    await data.TileView.AnimateMoveToStack(targetPos, 0.5f)
        .AsyncWaitForCompletion();

    int matchIdx = _stackModel.FindMatch();
    if (matchIdx >= 0) {
        await AnimateMatchRemoval(matchIdx);
        await _stackView.AnimateArrange(GetTileViews())
            .AsyncWaitForCompletion();
    }
}
```

## 3. Event Channel vs EventBus — Quyết Định Thiết Kế

| | EventBus (static) | Event Channel SO (chọn) |
|---|---|---|
| **Coupling** | Type-based | **SO reference** — thấp hơn |
| **Debug** | Code only | **Inspector visible** |
| **Scene** | Global | **Per-scene hoặc shared** |
| **Leak risk** | Cao (static) | **Thấp** (SO lifetime) |
| **Testing** | Mock khó | **Tạo test SO** |
| **Config** | Hardcoded | **Inspector** |

## 4. Singleton Strategy

```
📌 Chỉ giữ 3 Singleton (DontDestroyOnLoad):
   1. AudioService
   2. SaveService
   3. SceneService

📌 KHÔNG dùng Singleton cho:
   - GameController, BoardController, StackController (per-scene)
   - Tất cả Model (tạo bằng new, inject vào Controller)

📌 Event Channel SO:
   - Là assets, sống suốt vòng đời project
   - Không cần Singleton
   - Wire qua [SerializeField] trong Inspector
```

## 5. Performance Notes

| Mục | Ghi chú |
|---|---|
| **DOTween** | Luôn `SetAutoKill(true)`, `KillAll()` khi chuyển scene |
| **Event Channel** | Unsubscribe trong `OnDisable()` mọi lúc |
| **Sprite Atlas** | Đóng gói tất cả tile sprite vào 1 atlas |
| **Object Pooling** | TileView nên dùng pool |
| **GC Allocation** | Tránh `new List<>()` trong Update |
| **PlayerPrefs.Save()** | Batch save qua SaveService |

## 6. Pitfalls Thường Gặp

### 6.1 Event Channel reference null

```
⚠️ Quên wire Event Channel SO trong Inspector → NullReferenceException
📌 Giải pháp: Validate trong OnEnable hoặc dùng [Required] attribute
```

### 6.2 View references Model trực tiếp

```csharp
// ❌ SAI
public class TileView : MonoBehaviour {
    void Update() {
        if (tileModel.IsSelectable) { ... }  // View biết Model!
    }
}

// ✅ ĐÚNG: Controller push data
public class BoardController : MonoBehaviour {
    private void SyncView() {
        foreach (var tile in _boardModel.Tiles) {
            var view = GetViewForModel(tile);
            view.SetSelectable(tile.IsSelectable);
        }
    }
}
```

### 6.3 Controller quá "béo"

```
📌 Controller > 300 dòng → cần tách
   - GameController: chỉ quản lý phase + điều phối
   - KHÔNG chứa match-3 logic (đó là StackModel)
   - KHÔNG chứa overlap logic (đó là BoardModel)
```
