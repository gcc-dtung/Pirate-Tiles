## 1. Các Vấn Đề Trong Codebase Hiện Tại

### 1.1 Architectural Issues

| # | Vấn đề | File | Mức độ | Giải pháp MVC |
|---|---|---|---|---|
| 1 | **God class** — `Card.cs` vừa chứa data, logic, animation, input, VFX | `Card.cs` | 🔴 Nghiêm trọng | Tách thành `CardModel` + `CardView` + `CardVFXView` |
| 2 | **Singleton lạm dụng** — 8+ singleton, tạo tight coupling | Nhiều file | 🔴 Nghiêm trọng | Giảm còn 3–4 Service singleton, dùng DI cho phần còn lại |
| 3 | **Logic rải rác** — overlap check ở View, match logic ở StackLogic kèm animation | `CardOverlapChecker`, `StackLogic` | 🔴 Nghiêm trọng | Overlap logic → `BoardModel`, Match logic → `StackModel` |
| 4 | **Event đăng ký anonymous lambda** — không thể unsubscribe | `MagicPowerUp.cs` L77–80 | 🟡 Memory leak | Dùng named method hoặc EventBus typed |
| 5 | **Static mutable state** — `Board.cards`, `Board.currCardCount`, `HeartsSystem.hearts` | `Board.cs`, `HeartsSystem.cs` | 🟡 Khó test | Chuyển thành instance fields trong Model |
| 6 | **`FindGameObjectsWithTag` trong runtime** — chậm, fragile | `Board.cs` L33 | 🟡 Performance | BoardView quản lý danh sách CardView, inject vào BoardController |
| 7 | **`GameObject.Find` trong constructor** — race condition | `MagicPowerUp.cs`, `ShufflePowerUp.cs` | 🟡 Bug tiềm ẩn | Inject dependency qua constructor hoặc SerializeField |
| 8 | **PlayerPrefs rải rác** — SaveData gọi ở mọi nơi | Nhiều file | 🟡 Khó maintain | Tập trung qua `SaveService` |
| 9 | **Debug keys trong production** — phím A, S, R trong code | `HeartsSystem.cs`, `PlayerManager.cs` | 🟢 | Loại bỏ hoặc wrap trong `#if UNITY_EDITOR` |
| 10 | **DOTween không kill đúng cách** — có thể leak | Nhiều file | 🟡 | `.SetAutoKill(true)` + `OnDestroy() → DOKill()` trên tất cả View |

### 1.2 Code Smells cần fix

```
⚠️ Card.Awake() reset các flag global:
   GameModeManager.instance.isPaused = false;
   → Mỗi Card khi spawn sẽ force-reset pause state
   → Giải pháp: Chỉ GameController quản lý state flags

   GameEvents.OnMatchTilesDone += () => { ... };
   → Anonymous lambda KHÔNG BAO GIỜ unsubscribe được

⚠️ ShufflePowerUp chạy logic trong OnEnable():
```

## 2. Lưu Ý Kỹ Thuật Quan Trọng

### 2.1 Overlap Detection — Cơ chế mới

**Codebase cũ:** Dùng `Physics.OverlapBox` (3D physics) trên trục Z.

**Đề xuất MVC:** Chuyển sang **data-driven overlap** trong `BoardModel`:

```csharp
// Thay vì dùng Physics, tính overlap bằng dữ liệu grid
public class BoardModel {
    // Mỗi card biết vị trí grid + layer
    // Card ở layer cao hơn (Z nhỏ hơn) che card ở layer thấp

    public void UpdateSelectableStatus() {
        foreach (var card in _cards.Where(c => c.State == CardState.InBoard)) {
            bool isCovered = _cards.Any(other =>
                other.State == CardState.InBoard &&
                other.LayerIndex < card.LayerIndex && // Phía trên
                RectsOverlap(card.GridPosition, other.GridPosition)
            );
            card.IsSelectable = !isCovered;
        }
    }

    private bool RectsOverlap(Vector2Int posA, Vector2Int posB) {
        // Kiểm tra 2 card có overlap trên mặt phẳng XY không
        // Dựa trên grid position + card size
        return Mathf.Abs(posA.x - posB.x) < cardWidth &&
               Mathf.Abs(posA.y - posB.y) < cardHeight;
    }
}
```

**Ưu điểm:**
- ✅ Không cần Physics system (nhẹ hơn)
- ✅ Có thể unit test không cần Unity
- ✅ Deterministic (không phụ thuộc frame/physics update)

**Nếu vẫn muốn dùng Physics:**
- Đặt logic OverlapBox ở `BoardController` (không phải View)
- Kết quả trả về cho `BoardModel` cập nhật `IsSelectable`

### 2.2 DOTween Best Practices

```csharp
// ✅ ĐÚNG: Set AutoKill + Kill trong OnDestroy
public class CardView : MonoBehaviour {
    private Tween _currentTween;

    public Tween AnimateMove(Vector3 target, float dur) {
        _currentTween?.Kill();
        _currentTween = transform.DOMove(target, dur)
            .SetAutoKill(true)
            .OnComplete(() => _currentTween = null);
        return _currentTween;
    }

    private void OnDestroy() {
        _currentTween?.Kill();
    }
}

// ❌ SAI: Sequence không được track
transform.DOMove(...);  // Orphaned tween
DOTween.Sequence().Append(...);  // Không kill được
```

### 2.3 EventBus — Tránh Memory Leak

```csharp
// ✅ ĐÚNG: Subscribe trong OnEnable, Unsubscribe trong OnDisable
private void OnEnable() {
    EventBus.Subscribe<CardSelectedEvent>(OnCardSelected);
}
private void OnDisable() {
    EventBus.Unsubscribe<CardSelectedEvent>(OnCardSelected);
}

// ❌ SAI: Subscribe anonymous lambda
EventBus.Subscribe<CardSelectedEvent>(e => DoSomething(e));
// → Không thể Unsubscribe!
```

### 2.4 Async/Await với DOTween

```csharp
// Có thể dùng UniTask hoặc DOTween AsyncWaitForCompletion
// Giúp code tuần tự, dễ đọc hơn callback hell

private async void OnCardSelected(CardSelectedEvent evt) {
    // 1. Animation di chuyển
    await evt.CardView.AnimateMoveToStack(targetPos, 0.5f)
        .AsyncWaitForCompletion();

    // 2. Check match
    int matchIdx = _stackModel.FindMatch();
    if (matchIdx >= 0) {
        // 3. Animation match
        await AnimateMatchRemoval(matchIdx);

        // 4. Sắp xếp lại
        await _stackView.AnimateArrange(GetCardViews())
            .AsyncWaitForCompletion();
    }
}
```

> **Lưu ý:** `AsyncWaitForCompletion()` cần DOTween Pro hoặc UniTask bridge. Nếu không có, dùng callback chain.

## 5. Các Quyết Định Thiết Kế Quan Trọng

### 5.1 EventBus vs. C# Events vs. UnityEvent

| | C# Events (cũ) | EventBus (mới) | UnityEvent |
|---|---|---|---|
| **Coupling** | Trung bình (cần reference) | Thấp (type-based) | Thấp (Inspector) |
| **Type Safety** | ✅ | ✅ (generic) | ❌ |
| **Debug** | Khó | Dễ (centralized) | Dễ (Inspector) |
| **Performance** | Nhanh | Nhanh | Chậm hơn (reflection) |
| **Chọn** | — | ✅ **EventBus** | Cho UI button click |

### 5.2 Singleton Strategy

```
📌 Chỉ giữ 3 Singleton (DontDestroyOnLoad):
   1. AudioService
   2. SaveService
   3. SceneService

📌 Không dùng Singleton cho:
   - GameController (per-scene, inject via hierarchy)
   - BoardController (per-scene)
   - StackController (per-scene)
   - PowerUpController (per-scene)
   - Tất cả Model (tạo bằng new, inject vào Controller)
```

### 5.3 Data Persistence — Nâng cấp trong tương lai

```
📌 Hiện tại: PlayerPrefs (đơn giản, đủ dùng)
📌 Tương lai: Có thể chuyển sang JSON file hoặc SQLite
   → SaveService đóng vai trò abstraction layer
   → Chỉ cần thay đổi implementation trong SaveService
   → Không ảnh hưởng đến Model/Controller
```

## 6. Pitfalls Thường Gặp Khi Implement MVC trong Unity

### 6.1 View references Model trực tiếp

```csharp
// ❌ SAI: View đọc Model
public class CardView : MonoBehaviour {
    public void Update() {
        if (cardModel.IsSelectable) { ... }  // View biết Model!
    }
}

// ✅ ĐÚNG: Controller sync data cho View
public class BoardController : MonoBehaviour {
    private void SyncView() {
        foreach (var card in _boardModel.Cards) {
            var view = GetViewForModel(card);
            view.SetSelectable(card.IsSelectable);  // Controller push data
        }
    }
}
```

### 6.2 Controller quá "béo"

```
📌 Nếu một Controller > 300 dòng → cần tách:
   - GameController: chỉ quản lý phase + điều phối
   - KHÔNG chứa logic Match-3 (đó là StackModel)
   - KHÔNG chứa logic Overlap (đó là BoardModel)
   - KHÔNG chứa animation (đó là View)
```

### 6.3 Model dùng MonoBehaviour

```csharp
// ❌ SAI: Model kế thừa MonoBehaviour
public class BoardModel : MonoBehaviour {
    void Update() { ... }  // Model KHÔNG CẦN Update
}

// ✅ ĐÚNG: Pure C# class
public class BoardModel {
    // Không có Update, Start, Awake
    // Chỉ có methods được Controller gọi
}
```

## 7. Performance Notes

| Mục | Ghi chú |
|---|---|
| **DOTween** | Luôn `SetAutoKill(true)`, `KillAll()` khi chuyển scene |
| **Physics.OverlapBox** | Nếu dùng, gọi ít nhất có thể (chỉ khi card rời đi, không mỗi frame) |
| **FindGameObjectsWithTag** | Thay bằng BoardView quản lý `List<CardView>` |
| **PlayerPrefs.Save()** | Batch save, không gọi sau mỗi thay đổi nhỏ |
| **Sprite Atlas** | Đóng gói tất cả card sprite vào 1 atlas để giảm draw calls |
| **Object Pooling** | CardView nên dùng pool thay vì Instantiate/Destroy liên tục |
| **GC Allocation** | Tránh `new List<>()` trong Update/mỗi frame, dùng reusable buffer |
