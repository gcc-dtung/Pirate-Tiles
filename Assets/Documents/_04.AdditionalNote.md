## 1. Lưu Ý Kỹ Thuật Quan Trọng

### 1.1 Overlap Detection — Data-Driven

```csharp
public class BoardModel {
    public void UpdateSelectableStatus() {
        foreach (var tile in _tiles.Where(t => t.State == CardState.InBoard)) {
            bool isCovered = _tiles.Any(other =>
                other.State == CardState.InBoard &&
                other.LayerIndex < tile.LayerIndex &&
                RectsOverlap(tile.GridPosition, other.GridPosition)
            );
            tile.IsSelectable = !isCovered;
        }
        // Raise EventBus — nội bộ Model notification
        EventBus<BoardModelUpdatedEvent>.Raise(new BoardModelUpdatedEvent {
            RemainingTiles = RemainingTiles,
            SelectableTiles = _tiles.Count(t => t.IsSelectable)
        });
    }
}
```

### 1.2 DOTween Best Practices

```csharp
public class TileView : MonoBehaviour {
    private Tween _currentTween;

    public Tween AnimateMove(Vector3 target, float dur) {
        _currentTween?.Kill();
        _currentTween = transform.DOMove(target, dur)
            .SetAutoKill(true)
            .OnComplete(() => _currentTween = null);
        return _currentTween;
    }

    private void OnDestroy() {
        _currentTween?.Kill();
    }
}
```

### 1.3 Event Channel SO — Tránh Memory Leak

```csharp
// ✅ ĐÚNG: Subscribe trong OnEnable, Unsubscribe trong OnDisable
[SerializeField] private VoidEventChannelSO _gameWonChannel;

private void OnEnable() {
    _gameWonChannel.Subscribe(OnGameWon);
}

private void OnDisable() {
    _gameWonChannel.Unsubscribe(OnGameWon);
}

// ❌ SAI: Subscribe anonymous lambda
_gameWonChannel.Subscribe(() => DoSomething());
// → Không thể Unsubscribe!
```

### 1.4 EventBus — Tránh Memory Leak

```csharp
// ✅ ĐÚNG: Lưu binding reference, deregister trong OnDisable
private EventBinding<GamePhaseChangedEvent> _phaseBinding;

private void OnEnable() {
    _phaseBinding = new EventBinding<GamePhaseChangedEvent>(OnPhaseChanged);
    EventBus<GamePhaseChangedEvent>.Register(_phaseBinding);
}

private void OnDisable() {
    EventBus<GamePhaseChangedEvent>.Deregister(_phaseBinding);
}

// ❌ SAI: Không lưu binding reference
EventBus<GamePhaseChangedEvent>.Register(
    new EventBinding<GamePhaseChangedEvent>(OnPhaseChanged)
);
// → Không thể Deregister!
```

## 3. Hybrid Event System — Quyết Định Thiết Kế

### 3.1 Tại sao dùng cả hai?

| Vấn đề | Giải pháp |
|---|---|
| Model (pure C#) cần notify mà không reference MonoBehaviour | **EventBus** — static, type-based |
| Controller cần giao tiếp cross-layer, designer cần debug | **Event Channel SO** — Inspector visible |
| System-level events (scene load, save) không cần UI debug | **EventBus** — lightweight, global |
| Gameplay events cần wire per-scene | **Event Channel SO** — SO asset, [SerializeField] |

### 3.2 Bảng quyết định

```
📌 DÙNG EventBus khi:
   ✅ Pure C# context (Model layer)
   ✅ System-level, service-internal
   ✅ Tần suất cao, không cần Inspector debug
   ✅ Không cần per-scene configuration

📌 DÙNG Event Channel SO khi:
   ✅ Cross-layer communication
   ✅ Designer/QA cần theo dõi trong Inspector
   ✅ Cần wire khác nhau giữa các scene
   ✅ Lifecycle gắn với MonoBehaviour
```

### 3.3 So sánh chi tiết

| | EventBus (Static Generic) | Event Channel SO |
|---|---|---|
| **Coupling** | Type-based (thấp) | SO reference (rất thấp) |
| **Debug** | Code only | **Inspector visible** |
| **Config** | Hardcoded | **Inspector-configurable** |
| **Scene** | Global static | **Per-scene hoặc shared** |
| **Leak risk** | Cần clear (EventBusUtil) | Dễ kiểm soát (OnDisable) |
| **Testing** | Mock qua interface | Tạo test SO |
| **Pure C#** | ✅ Hỗ trợ | ❌ Cần MonoBehaviour |
| **Payload** | struct : IEvent | struct (EventData) |

```
📌 Chỉ giữ 3 Singleton (DontDestroyOnLoad):
   1. AudioService
   2. SaveService
   3. SceneService

📌 EventBus:
   - Static generic class, tự động khởi tạo qua EventBusUtil
   - Clear tự động khi exit Play Mode

📌 Event Channel SO:
   - Là assets, sống suốt vòng đời project
   - Không cần Singleton
   - Wire qua [SerializeField] trong Inspector
```

## 5. Performance Notes

| Mục | Ghi chú |
|---|---|
| **DOTween** | Luôn `SetAutoKill(true)`, `KillAll()` khi chuyển scene |
| **Event Channel SO** | Unsubscribe trong `OnDisable()` mọi lúc |
| **EventBus** | Deregister binding trong `OnDisable()`, EventBusUtil clear khi exit Play Mode |
| **Sprite Atlas** | Đóng gói tất cả tile sprite vào 1 atlas |
| **Object Pooling** | TileView nên dùng pool |
| **GC Allocation** | Tránh `new List<>()` trong Update. Event payload dùng struct để tránh GC |
| **PlayerPrefs.Save()** | Batch save qua SaveService |

## 6. Pitfalls Thường Gặp

### 6.1 Event Channel reference null
```
⚠️ Quên wire Event Channel SO trong Inspector → NullReferenceException
📌 Giải pháp: Validate trong OnEnable hoặc dùng [Required] attribute
```

### 6.2 EventBus binding không lưu reference
```
⚠️ Tạo binding inline mà không lưu → không thể Deregister → memory leak
📌 Giải pháp: Luôn lưu binding vào field: _binding = new EventBinding<T>(...)
```

### 6.3 View references Model trực tiếp
```csharp
// ❌ SAI
public class TileView : MonoBehaviour {
    void Update() {
        if (tileModel.IsSelectable) { ... }
    }
}

// ✅ ĐÚNG: Controller push data
public class BoardController : MonoBehaviour {
    private void SyncView() {
        foreach (var tile in _boardModel.Tiles) {
            var view = GetViewForModel(tile);
            view.SetSelectable(tile.IsSelectable);
        }
    }
}
```
