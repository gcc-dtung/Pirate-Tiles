# 🗺️ Hướng Dẫn Setup Scene Hierarchy — Pirate Tiles

> Tài liệu mô tả chi tiết cấu trúc Hierarchy cho từng Scene trong Unity Editor.
> Mỗi mục liệt kê: **GameObject cần tạo**, **Script cần gắn**, và **SerializeField cần kéo thả**.

---

## Prefab Dùng Chung: `GameManager` (DontDestroyOnLoad)

> Prefab này tồn tại xuyên suốt tất cả Scene. Tạo 1 lần, đặt vào Scene đầu tiên load.

```
GameManager (GameObject)
├── [Script] GameManager.cs          ← Singleton, DontDestroyOnLoad
├── [Script] AudioService.cs
├── [Script] SaveService.cs
└── [Script] SceneService.cs
```

**Không có SerializeField View nào.** Các Service này hoạt động độc lập qua EventBus.

## Scene 2: `Map` — Chọn Level

```
Map (Scene)
│
├── 📷 Main Camera
├── 🎨 Background
│
├── Canvas (Screen Space - Overlay)
│   ├── TopBar
│   │   ├── HeartsPanel
│   │   │   └── [Script] HeartsView.cs
│   │   └── CoinsPanel
│   │       └── [Script] CoinsView.cs
│   │
│   ├── MapScrollView (ScrollRect)
│   │   └── Content (Vertical/Grid Layout)
│   │       └── [Script] MapView.cs
│   │           ├── _levelButtonPrefab ← Asset: LevelButton.prefab
│   │           └── _contentContainer ← chính Content Transform này
│   │
│   └── SettingButton (Button) → Mở SettingPanel
│
└── EventSystem
```

**Lưu ý:** `HeartsController` và `CoinsController` nên nằm trong `GameManager` (DontDestroyOnLoad) nếu muốn dữ liệu tồn tại cross-scene. Hoặc dùng `SaveService` để load lại.

## Scene 3: `Loading` — Màn Hình Chờ

```
Loading (Scene)
│
├── 📷 Main Camera
│
└── Canvas
    └── LoadingPanel
        └── [Script] LoadingView.cs
```

**SceneService** sẽ load scene này trước, sau đó async load scene đích và gọi `UpdateProgress()`.
